# Plan

## Issue 1: Navigation bar alignment

The dark/light mode icon in the navigation bar is not aligned on the same vertical axis as the "Educational AI Agent" label and the navigation tabs.

Proposed solution:
- Modify the navigation bar in `client/src/App.tsx` to group the theme toggle button with the brand label on the left side.
- Change the structure of the nav-wrapper inner flex container to have two children:
    1. A div containing the brand label and the theme toggle button (both in a flex items-center gap-2 container)
    2. A div containing the navigation tabs (Document Viewer, Saves, Settings) in a flex items-center gap-1 container
- This ensures that the brand label and the theme toggle button are vertically aligned (as they are in the same flex container) and the navigation tabs are also vertically aligned in their own container, and the two containers are vertically aligned by the parent flex container's `items-center`.

## Issue 2: ReferenceError: theme is not defined in DocumentViewer

The `DocumentViewer` component attempts to use a `theme` variable that is not defined in its scope.

Proposed solution:
- Import the `useTheme` hook from `./contexts/ThemeContext` in `client/src/pages/DocumentViewer.tsx`.
- Inside the `DocumentViewer` component, call `const { theme } = useTheme();` to access the current theme value.
- Replace the usage of the undefined `theme` variable with the one obtained from the hook.
