# Fix: Login page placeholder text not disappearing on input

## Root Cause

Materialize CSS JS is **not loaded** in the app (`index.html` only loads the CSS CDN, no JS CDN). Materialize's floating label behavior requires its JavaScript to dynamically add/remove the `active` CSS class on `<label>` elements when inputs have values or receive focus. Without JS, labels stay in their placeholder position permanently — they never float up, so the text appears to "not disappear" when the user types.

## Plan (CSS-only fix, no JS library needed)

### File 1: `client/src/pages/Login.tsx`

Add `placeholder=" "` (a single space) to both `<input>` elements. This enables the CSS `:not(:placeholder-shown)` pseudo-class, which detects whether the input has a non-empty value.

**Line 43** — username input:
```tsx
<input id="username" type="text" value={username} onChange={e => setUsername(e.target.value)} placeholder=" " required />
```

**Line 52** — password input:
```tsx
<input id="password" type="password" value={password} onChange={e => setPassword(e.target.value)} placeholder=" " required />
```

### File 2: `client/src/index.css`

Add CSS rules at the end of the file to float the label when the input is focused OR has a value:

```css
.input-field input:focus + label,
.input-field input:not(:placeholder-shown) + label {
  transform: translateY(-14px) scale(0.8);
  transform-origin: 0 0;
}
```

- `input:focus + label` — floats label on focus
- `input:not(:placeholder-shown) + label` — floats label when input has a value

## Why this works

- `:placeholder-shown` is `true` when input is empty (space placeholder is visible), `false` when input has user-entered text
- `:not(:placeholder-shown)` inverts this — true when input has a value
- Combined with `:focus`, the label floats in both cases (focused OR has value)
- No additional JS dependencies needed
- Zero runtime cost — pure CSS
- Matches Materialize CSS's existing transform values (`translateY(-14px) scale(0.8)`)
